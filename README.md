# キャリア概要
私の大学時代から現在に至るまでの流れ、使用技術、今後の展望をまとめたものです。

## 🎓 大学、大学院時代（2015.04〜2021.09）
- 日本の私立大学で食品科学・微生物学を専攻
- 発酵、衛生管理、微生物の挙動について学習
- 大学4年時、朝10時から夜23時まで研究活動を半年継続する
  - 研究 + 研究室内の雑用 + 後輩への教育
- 大学院進学後、研究を継続するも途中で休学

### 🔍 休学の背景
- 朝から夜21時まで研究作業を続ける日々の中でスランプに陥る
- 自然現象は検証困難なことが多く、仮説検証したとしても正しい結論なのか疑問が増す
- 研究への意欲が低下したことから、自分により適したキャリアを模索するため、就職活動を開始

## 🚪 就職活動と制度的障壁（2020.07〜2022.04）

大学院休学後、公務員試験を受験。その後、民間企業への就職活動を開始し、200社以上に応募。

- 小売業、施工会社、IT企業など幅広い業種に応募
- 2021年9月より派遣勤務を開始し、月収10万円程度でアパートにて独立生活を開始  
  - 経済的に厳しい状況の中でも、都内のIT企業に絞って就職活動を継続

---

## 🎯 職種選択の動機

### 🛍️ 小売業
- 成果が目に見えやすく、日々の業務がモチベーションにつながる
- 接客や売上など、直接的なフィードバックが得られる

### 🏗️ 施工会社
- 建築物という形で、自分の仕事の成果が残る
- プロジェクトの達成感が強く、やりがいを感じやすい

### 💻 IT業界
- インターネット環境があれば、どこでも柔軟に情報収集や自宅の環境で検証が可能
- 人が作ったシステムを使用するものであるため、完全に理屈に合わない事象が起きにくい
- 技術が日々進化するため、飽きることがなく継続的に学べる
- 自身の手を動かして試すのが好きな性格と合っていると感じた

### ☁️ クラウドオペレーションエンジニア（AWS中心）（2023.09〜現在）

- VBAによる書類作成、AWSリソースのパラメータシートの作成業務の自動化
- QuickSightとLambdaでAWSコスト計算・RDB格納
- CloudFormationによる構成管理・更新
  - DynamoDB, S3, CloudWatch, ECS, Lambda

- Python, PowerShell, AWS Batchによる自動化スクリプト作成
  - AWSリソースの設定値取得(EC2, S3, SNS, ECS, ECR, EventBridge, Lambda, DynamoDBなど20種類以上のサービス)
  - 自動CloudFormationスタック作成

  - AWS CLI未導入端末のEventViewerログを収集・共有・CloudWatchに連携し、エラー検知を自動化
  
- 使用技術：CloudWatch, Lambda, S3, EC2, SNS, Health Dashboard, CloudFormation, SecurityHub, DynamoDB
